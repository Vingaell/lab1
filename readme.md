# 🚗 Car Object Detection

## 📌 Описание проекта

В рамках работы решается задача детекции автомобилей на изображениях с использованием моделей YOLO, а также собственной имплементированной модели.

---

## 📊 Выбор датасета и метрик

В качестве набора данных был выбран датасет:  
https://www.kaggle.com/datasets/sshikamaru/car-object-detection

Данный датасет содержит изображения с размеченными bounding box’ами автомобилей и соответствует реальной задаче детекции объектов. Выбор обусловлен практической значимостью: обнаружение автомобилей широко применяется в системах видеонаблюдения, анализе дорожного трафика и автономных транспортных системах.

Дополнительно датасет уже содержит разметку, совместимую с форматом YOLO, что упрощает подготовку данных и позволяет сосредоточиться на обучении моделей и сравнении результатов.

В качестве метрик качества были выбраны:

**Precision** — показывает долю правильных предсказаний среди всех предсказанных объектов. Чем выше значение, тем меньше ложных срабатываний.

**Recall** — показывает, какую долю реальных объектов модель смогла обнаружить. Чем выше значение, тем меньше пропущенных объектов.

**mAP50** — средняя точность детекции при IoU = 0.5. Показывает, насколько хорошо модель находит объекты при относительно мягком критерии совпадения границ.

**mAP50-95** — усреднённая метрика по диапазону IoU от 0.5 до 0.95. Учитывает качество локализации объектов и является наиболее строгой и показательной метрикой.

## 📁 Содержимое проекта

В Jupyter Notebook реализовано:

- обучение моделей YOLO (YOLO11n, YOLO11s)
- улучшение бейзлайна (аугментации, заморозка слоёв)
- сравнение моделей по метрикам
- имплементация собственной модели
- обучение и оценка имплементированной модели
- итоговое сравнение результатов

## ⚙️ Установка и запуск

Проект выполнялся на windows 10. Ну других системах успешный запуск с этой инструкцией не гарантирован.

### 1. Установка Python

Установите Python версии **3.12.9**

---

### 2. Создание виртуального окружения

```bash
python -m venv venv

Активация:

```bash
venv\Scripts\activate
```

### 3. Установка зависимостей

Установка PyTorch с поддержкой CUDA:

```bash
pip install torch==2.3.1 torchvision==0.18.1 torchaudio==2.3.1 --index-url https://download.pytorch.org/whl/cu121
```

Установка остальных зависимостей:

```bash
pip install -r requirements.txt
```

### 4. Подготовка данных

Датасет уже загружен на гит, нужно просто скачать весь код с гита и разместить проект по этому адресу на персональном компьютере:

D:\python\II - где II - папка с проектом.

Если по какой-либо причине возможно положить проект в эту папку нет, необходимо изменить пути в коде на удобные. 

Вот место, которое нужно изменить:
```bash
yaml_text = """
train: D:/python/II/dataset/images/train
val: D:/python/II/dataset/images/val

names:
  0: car
"""
with open("data.yaml", "w") as f:

    f.write(yaml_text)
```
Менять можно только "D:/python/II/". Все папки дальше создаются автоматически в ходе выполнения кода.

### 5. Запуск проекта

```bash
jupyter notebook
```

Откройте .ipynb файл и выполняйте ячейки последовательно.

### 📈 Результаты

Модели YOLO показали высокое качество по всем выбранным метрикам.

Имплементированная модель продемонстрировала значительно более слабые результаты, что связано с упрощённой архитектурой и отсутствием полноценных механизмов детекции.

Наилучшие результаты достигаются при использовании более мощной модели и комбинации техник обучения, таких как аугментации и заморозка слоёв.
